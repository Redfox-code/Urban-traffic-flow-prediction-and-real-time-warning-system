## 2. 数据字典

### 2.1 users — 用户表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 用户唯一ID |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL | — | 登录用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | — | bcrypt哈希后的密码 |
| `role` | ENUM('admin','analyst') | NOT NULL | 'analyst' | 角色：admin=交通管理者, analyst=数据分析员 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE | CURRENT_TIMESTAMP | 最后更新时间 |

**索引**：`UNIQUE INDEX idx_username (username)`

### 2.2 traffic_sections — 路段表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 路段唯一ID |
| `name` | VARCHAR(100) | NOT NULL | — | 路段名称（如"长安街东段"） |
| `capacity` | INT | NOT NULL | — | 通行能力（veh/h） |
| `length` | DECIMAL(6,2) | NOT NULL | — | 路段长度（km） |
| `max_speed` | INT | NOT NULL | 60 | 最高限速（km/h） |
| `coordinates` | JSON | NOT NULL | — | 路段起终点坐标 `{"start":[lng,lat],"end":[lng,lat],"midpoints":[[lng,lat],...]}` |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | — |
| `updated_at` | DATETIME | ON UPDATE | CURRENT_TIMESTAMP | — |

**索引**：`INDEX idx_section_name (name)`

### 2.3 traffic_detectors — 检测器表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 检测器唯一ID |
| `section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 所属路段 |
| `type` | ENUM('e2','induction_loop') | NOT NULL | 'e2' | 检测器类型 |
| `position` | DECIMAL(6,2) | NOT NULL | — | 距路段起点距离（km） |
| `status` | ENUM('active','inactive','maintenance') | NOT NULL | 'active' | 设备状态 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | — |

**索引**：`INDEX idx_detector_section (section_id)`, `INDEX idx_detector_status (status)`

### 2.4 traffic_records — 流量记录表（核心数据表）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 记录唯一ID |
| `section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 所属路段 |
| `detector_id` | INT | FK→traffic_detectors.id, NOT NULL | — | 采集检测器 |
| `vehicle_count` | INT | NOT NULL | — | 统计间隔内通过车辆数 |
| `avg_speed` | DECIMAL(5,2) | NOT NULL | — | 平均车速（km/h） |
| `occupancy` | DECIMAL(5,2) | NOT NULL | — | 时间占有率（%） |
| `timestamp` | DATETIME | NOT NULL | — | 数据采集时间戳 |

**索引**：
- `INDEX idx_record_section (section_id)`
- `INDEX idx_record_timestamp (timestamp)`
- `INDEX idx_record_section_time (section_id, timestamp)` — 复合索引，最常用查询
- 分区策略：按月RANGE分区（预估日增5760条记录，月增17万条）

**预估数据量**：40检测器 × 24小时 × 4次/小时 = 3,840条/天（15min采样），实际SUMO仿真输出约10倍于此

### 2.5 prediction_results — 预测结果表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 预测结果ID |
| `section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 预测路段 |
| `model_type` | ENUM('KNN','RF') | NOT NULL | — | 使用的预测模型 |
| `predicted_flow` | DECIMAL(8,2) | NOT NULL | — | 预测车流量（veh/h） |
| `confidence_lower` | DECIMAL(8,2) | NOT NULL | — | 置信区间下界 |
| `confidence_upper` | DECIMAL(8,2) | NOT NULL | — | 置信区间上界 |
| `prediction_horizon` | INT | NOT NULL | 15 | 预测窗口（分钟） |
| `timestamp` | DATETIME | NOT NULL | — | 预测生成时间 |

**索引**：`INDEX idx_pred_section_time (section_id, timestamp)`

### 2.6 warning_events — 预警事件表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 预警事件ID |
| `section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 预警路段 |
| `level` | ENUM('WARNING','CRITICAL') | NOT NULL | — | 预警等级 |
| `message` | VARCHAR(500) | NOT NULL | — | 预警描述 |
| `trigger_flow` | DECIMAL(8,2) | NOT NULL | — | 触发预警时的流量值 |
| `threshold` | DECIMAL(8,2) | NOT NULL | — | 触发阈值 |
| `is_resolved` | BOOLEAN | NOT NULL | FALSE | 是否已解除 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 触发时间 |
| `resolved_at` | DATETIME | NULL | NULL | 解除时间 |

**索引**：`INDEX idx_warning_section (section_id)`, `INDEX idx_warning_resolved (is_resolved, created_at)`

### 2.7 route_records — 路径规划记录表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 记录ID |
| `user_id` | INT | FK→users.id, NOT NULL | — | 查询用户 |
| `origin_section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 起点路段 |
| `dest_section_id` | INT | FK→traffic_sections.id, NOT NULL | — | 终点路段 |
| `route_path` | JSON | NOT NULL | — | 路径节点序列 `[{section_id, lng, lat}, ...]` |
| `total_distance` | DECIMAL(8,2) | NOT NULL | — | 总距离（km） |
| `estimated_time` | INT | NOT NULL | — | 预估时间（分钟） |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | — |

**索引**：`INDEX idx_route_user (user_id, created_at)`

### 2.8 system_logs — 系统日志表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 日志ID |
| `user_id` | INT | FK→users.id, NULL | NULL | 操作人（NULL=匿名操作） |
| `action` | VARCHAR(100) | NOT NULL | — | 操作类型（如login, query_prediction, resolve_warning） |
| `details` | JSON | NULL | NULL | 操作详情 |
| `ip_address` | VARCHAR(45) | NOT NULL | — | 客户端IP |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | — |

**索引**：`INDEX idx_log_user (user_id)`, `INDEX idx_log_time (created_at)`

## 3. DDL 参考脚本

```sql
-- ============================================
-- 城市交通流量预测与实时预警系统 — DDL v1.0
-- 基于：D3-T01 (Leader) + D3-T05 (本文档)
-- 数据库：MySQL 8.x
-- ============================================

CREATE DATABASE IF NOT EXISTS traffic_prediction
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE traffic_prediction;

-- 1. 用户表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role ENUM('admin', 'analyst') NOT NULL DEFAULT 'analyst',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE INDEX idx_username (username)
) ENGINE=InnoDB;

-- 2. 路段表
CREATE TABLE traffic_sections (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    capacity INT NOT NULL COMMENT '通行能力(veh/h)',
    length DECIMAL(6,2) NOT NULL COMMENT '长度(km)',
    max_speed INT NOT NULL DEFAULT 60 COMMENT '限速(km/h)',
    coordinates JSON NOT NULL COMMENT '起终点+中间点坐标',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_section_name (name)
) ENGINE=InnoDB;

-- 3. 检测器表
CREATE TABLE traffic_detectors (
    id INT AUTO_INCREMENT PRIMARY KEY,
    section_id INT NOT NULL,
    type ENUM('e2', 'induction_loop') NOT NULL DEFAULT 'e2',
    position DECIMAL(6,2) NOT NULL COMMENT '距起点距离(km)',
    status ENUM('active', 'inactive', 'maintenance') NOT NULL DEFAULT 'active',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    INDEX idx_detector_section (section_id),
    INDEX idx_detector_status (status)
) ENGINE=InnoDB;

-- 4. 流量记录表（核心数据表 — 数据量最大）
CREATE TABLE traffic_records (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    section_id INT NOT NULL,
    detector_id INT NOT NULL,
    vehicle_count INT NOT NULL COMMENT '通过车辆数',
    avg_speed DECIMAL(5,2) NOT NULL COMMENT '平均车速(km/h)',
    occupancy DECIMAL(5,2) NOT NULL COMMENT '时间占有率(%)',
    timestamp DATETIME NOT NULL COMMENT '采集时间',
    FOREIGN KEY (section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    FOREIGN KEY (detector_id) REFERENCES traffic_detectors(id) ON DELETE CASCADE,
    INDEX idx_record_section (section_id),
    INDEX idx_record_timestamp (timestamp),
    INDEX idx_record_section_time (section_id, timestamp)
) ENGINE=InnoDB;

-- 5. 预测结果表
CREATE TABLE prediction_results (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    section_id INT NOT NULL,
    model_type ENUM('KNN', 'RF') NOT NULL,
    predicted_flow DECIMAL(8,2) NOT NULL COMMENT '预测流量(veh/h)',
    confidence_lower DECIMAL(8,2) NOT NULL,
    confidence_upper DECIMAL(8,2) NOT NULL,
    prediction_horizon INT NOT NULL DEFAULT 15 COMMENT '预测窗口(min)',
    timestamp DATETIME NOT NULL COMMENT '预测生成时间',
    FOREIGN KEY (section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    INDEX idx_pred_section_time (section_id, timestamp)
) ENGINE=InnoDB;

-- 6. 预警事件表
CREATE TABLE warning_events (
    id INT AUTO_INCREMENT PRIMARY KEY,
    section_id INT NOT NULL,
    level ENUM('WARNING', 'CRITICAL') NOT NULL,
    message VARCHAR(500) NOT NULL,
    trigger_flow DECIMAL(8,2) NOT NULL COMMENT '触发流量',
    threshold DECIMAL(8,2) NOT NULL COMMENT '触发阈值',
    is_resolved BOOLEAN NOT NULL DEFAULT FALSE,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    resolved_at DATETIME DEFAULT NULL,
    FOREIGN KEY (section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    INDEX idx_warning_section (section_id),
    INDEX idx_warning_resolved (is_resolved, created_at)
) ENGINE=InnoDB;

-- 7. 路径规划记录表
CREATE TABLE route_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    origin_section_id INT NOT NULL,
    dest_section_id INT NOT NULL,
    route_path JSON NOT NULL COMMENT '路径节点序列',
    total_distance DECIMAL(8,2) NOT NULL COMMENT '总距离(km)',
    estimated_time INT NOT NULL COMMENT '预估时间(min)',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (origin_section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    FOREIGN KEY (dest_section_id) REFERENCES traffic_sections(id) ON DELETE CASCADE,
    INDEX idx_route_user (user_id, created_at)
) ENGINE=InnoDB;

-- 8. 系统日志表
CREATE TABLE system_logs (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id INT DEFAULT NULL,
    action VARCHAR(100) NOT NULL COMMENT '操作类型',
    details JSON DEFAULT NULL COMMENT '操作详情',
    ip_address VARCHAR(45) NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_log_user (user_id),
    INDEX idx_log_time (created_at)
) ENGINE=InnoDB;
```

## 5. 测试用例草案（供D7阶段参考）

| 用例ID | 测试目标 | 输入 | 期望输出 |
|--------|---------|------|---------|
| TC-DB-01 | 用户唯一性 | INSERT相同username两次 | 第二次报Duplicate entry |
| TC-DB-02 | 外键约束 | INSERT traffic_record with section_id=999 | Foreign key constraint fail |
| TC-DB-03 | CASCADE删除 | DELETE traffic_section WHERE id=1 | 关联detector/record/prediction/warning全部级联删除 |
| TC-DB-04 | 默认值 | INSERT user不指定role | role='analyst' |
| TC-DB-05 | JSON字段 | INSERT coordinates非法JSON | JSON_VALID检查失败 |
| TC-DB-06 | 时间戳自动更新 | UPDATE user的username | updated_at自动更新 |
| TC-DB-07 | 复合索引命中 | SELECT * FROM traffic_records WHERE section_id=1 AND timestamp BETWEEN ... | EXPLAIN显示使用idx_record_section_time |
| TC-DB-08 | ENUM约束 | INSERT warning_event level='INFO' | 报错，level只能是WARNING或CRITICAL |
