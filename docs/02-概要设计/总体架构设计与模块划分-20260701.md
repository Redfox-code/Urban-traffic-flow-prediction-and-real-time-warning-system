## 1. 架构概览

### 1.1 架构模式

采用 **前后端分离 + MVC三层架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                       前端展示层 (Vue 3)                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐  │
│  │ 监控大屏  │ │ 预测看板  │ │ 预警管理  │ │  路径规划      │  │
│  │(FE-Main) │ │(FE-Main) │ │(FE-Map)  │ │  (FE-Main)    │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────────┘  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │         高德地图 + WebSocket (FE-Map)                 │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTP REST + WebSocket
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                      后端服务层 (Flask)                       │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐  │
│  │ 认证模块  │ │ 流量API  │ │ 预测API  │ │  预警服务      │  │
│  │ JWT Auth │ │ RESTful  │ │ RESTful  │ │  SocketIO     │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐  │
│  │ 路段API  │ │ 路径规划  │ │ 统计API  │ │  Celery任务    │  │
│  │ RESTful  │ │ Dijkstra │ │ RESTful  │ │  定时重训练    │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────────┘  │
└──────────────────────┬──────────────────────────────────────┘
                       │ SQLAlchemy ORM
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                       数据层                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐  │
│  │  MySQL   │ │  Redis   │ │  SUMO    │ │  模型文件      │  │
│  │ 业务数据  │ │ 队列/缓存 │ │ 仿真引擎  │ │  joblib .pkl  │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **关注点分离** | 路由层(Controller) → 服务层(Service) → 数据层(Model)，每层只做自己的事 |
| **接口先行** | 模块间通过明确的API契约通信，不假设实现细节 |
| **开闭原则** | 预测模型通过策略模式抽象基类，新增模型不影响现有代码 |
| **向前兼容** | API版本化（/api/v1/），后续升级不影响现有前端 |

## 2. 六大功能模块划分

### 2.1 模块清单

| 模块ID | 模块名称 | 负责Agent | 技术栈 | 优先级 |
|--------|---------|----------|--------|--------|
| M1 | 数据采集模块 | Agent-Algorithm | SUMO + TraCI + Pandas | P0 |
| M2 | 流量预测模块 | Agent-Algorithm | Scikit-learn(KNN+RF) + joblib | P0 |
| M3 | 实时监控模块 | Agent-Frontend-Main + Agent-Frontend-Map | Vue 3 + ECharts + 高德地图 | P0 |
| M4 | 拥堵预警模块 | Agent-Lead(后端) + Agent-Frontend-Map(前端) | Flask-SocketIO + 规则引擎 | P0 |
| M5 | 路径规划模块 | Agent-Lead | Dijkstra算法 + NetworkX | P1 |
| M6 | 系统管理模块 | Agent-Lead | Flask-JWT-Extended + 日志 | P1 |

### 2.2 模块间接口契约

#### M1 → M2（数据采集 → 预测模块）

```
M1输出：Pandas DataFrame
  字段：timestamp, section_id, vehicle_count, avg_speed, occupancy, hour, day_of_week, is_holiday
  格式：CSV文件 + 内存DataFrame传递
  位置：algorithm/data/processed/

M2输入：同上DataFrame
M2输出：prediction_result = {section_id, predicted_flow, confidence_interval, timestamp}
```

#### M2 → M4（预测 → 预警模块）

```
M2输出：预测结果（通过Agent-Lead的predict API）
M4输入：predict API返回的预测值 + 阈值配置
  触发条件：predicted_flow > section.capacity * 0.85 → WARNING
            predicted_flow > section.capacity * 0.95 → CRITICAL
```

#### M3 → M4（监控 → 预警联动）

```
前端地图点击路段 → 触发预警规则检查 → WebSocket推送预警弹窗
事件名：warning:new, warning:update, warning:resolve
```

#### 所有模块 → M6（认证贯通）

```
JWT Token在请求头传递：Authorization: Bearer <token>
角色校验：
  - 交通管理者(admin)：所有页面可访问
  - 数据分析员(analyst)：预测看板+历史数据，无权限修改配置
```
