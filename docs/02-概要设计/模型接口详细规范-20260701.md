## 1. predict_flow 完整 Schema

### 1.1 函数签名

```python
def predict_flow(
    section_id: int,
    recent_data: pd.DataFrame,
    model_type: str = "RF",
    horizon: int = 15
) -> dict
```

### 1.2 输入 Schema

#### recent_data DataFrame

| 列名 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `timestamp` | datetime | 是 | 采样时间戳 | `2026-07-01T10:00:00` |
| `vehicle_count` | int | 是 | 通过车辆数 | `85` |
| `avg_speed` | float | 是 | 平均车速(km/h) | `42.5` |
| `occupancy` | float | 是 | 时间占有率(%) | `35.2` |
| `hour` | int | 是 | 小时(0-23) | `10` |
| `day_of_week` | int | 是 | 周几(0=Mon) | `2` |
| `is_weekend` | int | 是 | 是否周末(0/1) | `0` |
| `is_peak` | int | 是 | 是否高峰(0/1) | `1` |

**最小行数**：4行（用于构造t-1/t-2/t-3滞后特征 + 滚动统计）

**约束**：
- 所有行必须是同一section_id的连续时间序列
- 时间间隔15min，不允许有缺失时间窗口
- occupancy范围[0, 100]
- avg_speed ≥ 0

### 1.3 输出 Schema

```json
{
  "section_id": 1,
  "model": "RF",
  "horizon": 15,
  "predicted_flow": 120.5,
  "confidence_interval": {
    "lower": 110.2,
    "upper": 130.8
  },
  "predictions": [
    {"timestamp": "2026-07-01T10:30:00", "predicted_flow": 118.0},
    {"timestamp": "2026-07-01T10:45:00", "predicted_flow": 122.5},
    {"timestamp": "2026-07-01T11:00:00", "predicted_flow": 121.0}
  ],
  "generated_at": "2026-07-01T10:15:00"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `section_id` | int | 路段ID |
| `model` | string | 模型类型：`KNN` / `RF` |
| `horizon` | int | 预测窗口(min)：5/15/30 |
| `predicted_flow` | float | 预测流量(veh/h)，predictions数组第一个值的整数表示 |
| `confidence_interval` | object | 95%置信区间（RF：预测方差；KNN：邻居值范围） |
| `predictions` | array | 预测时间序列，长度=horizon/5（5min粒度） |
| `generated_at` | string | 预测生成时间(ISO 8601) |

### 1.4 错误处理

| 错误场景 | 返回 |
|---------|------|
| recent_data行数 < 4 | `422: "数据不足：至少需要4个连续时间窗口"` |
| model_type不在[KNN,RF] | `400: "不支持的模型类型：{model_type}"` |
| horizon不在[5,15,30] | `400: "不支持的预测窗口：{horizon}"` |
| 模型文件未加载 | `500: "模型未初始化，请检查服务启动日志"` |
| 数据包含NaN/Inf | `422: "输入数据包含无效值"` |

## 3. 模型文件规范

### 3.1 文件命名

```
backend/saved_models/
├── knn_predictor_{YYYYMMDD}.pkl
├── rf_predictor_{YYYYMMDD}.pkl
└── scaler_{YYYYMMDD}.pkl
```

日期为训练日期。保留最近3个版本，旧版本自动归档。

### 3.2 文件内容

每个 `.pkl` 文件包含完整的Scikit-learn Pipeline对象：

```python
# knn_predictor_20260701.pkl 内容
{
    "model": KNeighborsRegressor(n_neighbors=5, weights='distance', metric='manhattan'),
    "feature_names": ["vehicle_count", "avg_speed", ...],
    "scaler_mean": [...],
    "scaler_std": [...],
    "training_date": "2026-07-01",
    "metrics": {"MAE": 12.3, "RMSE": 18.5, "MAPE": 15.2, "R2": 0.78},
    "training_samples": 3840
}
```

### 3.3 加载方式

```python
# 单例模式 — 应用启动时加载一次（对应禁忌#1）
# backend/app/services/prediction_service.py

class PredictionService:
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._load_models()
        return cls._instance
    
    def _load_models(self):
        import glob, joblib
        knn_files = sorted(glob.glob("backend/saved_models/knn_predictor_*.pkl"))
        rf_files = sorted(glob.glob("backend/saved_models/rf_predictor_*.pkl"))
        
        self.knn = joblib.load(knn_files[-1]) if knn_files else None
        self.rf = joblib.load(rf_files[-1]) if rf_files else None
```

## 5. 数据管道完整配置

```yaml
# algorithm/pipeline_config.yaml
pipeline:
  simulation:
    duration: 3600          # 仿真时长(s)
    sampling_interval: 900  # 采样间隔(s) = 15min
  
  cleaning:
    missing_strategy: ffill  # 前值填充 → 中位数填充
    outlier_method: iqr      # IQR方法
    outlier_threshold: 1.5   # Q1-1.5*IQR, Q3+1.5*IQR
  
  features:
    lag_steps: [1, 2, 3]     # 滞后步数
    rolling_window: 4        # 滚动窗口(1h)
    time_features: [hour, day_of_week, is_weekend, is_peak]
  
  training:
    test_split_ratio: 0.2
    cv_folds: 5
    random_seed: 42
```
