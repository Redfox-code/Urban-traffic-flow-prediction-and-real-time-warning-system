## 1. 通用约定

### 1.1 基础信息

| 项 | 值 |
|----|-----|
| Base URL | `http://localhost:5000/api/v1` |
| 认证方式 | JWT Bearer Token |
| Content-Type | `application/json` |
| 字符编码 | UTF-8 |

### 1.2 统一返回格式

```json
{
  "code": 200,
  "data": { ... },
  "message": "ok"
}
```

### 1.3 分页格式

请求：`?page=1&page_size=20`（默认page=1, page_size=20, max=100）

```json
{
  "code": 200,
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "page_size": 20,
    "total_pages": 5
  }
}
```

### 1.4 错误码

| code | 含义 | 触发场景 |
|------|------|---------|
| 200 | 成功 | — |
| 201 | 创建成功 | POST返回 |
| 400 | 参数错误 | 缺少必填字段/格式不符 |
| 401 | 未认证 | Token缺失/过期/无效 |
| 403 | 无权限 | 角色不匹配（analyst访问admin路由） |
| 404 | 资源不存在 | ID不存在 |
| 409 | 资源冲突 | 用户名已存在 |
| 422 | 业务校验失败 | 预测参数不合理/路径不可达 |
| 500 | 服务器内部错误 | 未预期异常 |

## 2. 认证模块 `/api/v1/auth`

### 2.1 POST /auth/login

```
登录
```

**Request**：
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**Response (200)**：
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "admin"
    },
    "expires_in": 86400
  },
  "message": "登录成功"
}
```

**Error (401)**：
```json
{"code": 401, "data": null, "message": "用户名或密码错误"}
```

### 2.2 POST /auth/register

```
注册（仅admin可创建analyst账号）
```

**Request**：
```json
{
  "username": "analyst_wang",
  "password": "pass123",
  "role": "analyst"
}
```

**Response (201)**：
```json
{"code": 201, "data": {"id": 2, "username": "analyst_wang", "role": "analyst"}, "message": "注册成功"}
```

**Error (409)**：
```json
{"code": 409, "data": null, "message": "用户名已存在"}
```

### 2.3 POST /auth/refresh

```
刷新Token（需在旧Token过期前调用）
```

**Header**：`Authorization: Bearer <old_token>`

**Response (200)**：
```json
{"code": 200, "data": {"token": "new_token...", "expires_in": 86400}, "message": "Token已刷新"}
```

## 5. 预测模块 `/api/v1/predict`

### 5.1 GET /predict/forecast

```
短时流量预测
```

**Query**：`?section_id=1&horizon=15&model=RF`

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| section_id | int | 是 | — | 路段ID |
| horizon | int | 否 | 15 | 预测窗口(min)，可选5/15/30 |
| model | string | 否 | RF | 模型类型，KNN/RF |

**Response (200)**：
```json
{
  "code": 200,
  "data": {
    "section_id": 1,
    "section_name": "主干道-南北1",
    "model": "RF",
    "horizon": 15,
    "predicted_flow": 120.5,
    "confidence_interval": {"lower": 110.2, "upper": 130.8},
    "predictions": [
      {"timestamp": "2026-07-01T10:30:00", "predicted_flow": 118.0},
      {"timestamp": "2026-07-01T10:45:00", "predicted_flow": 122.5},
      {"timestamp": "2026-07-01T11:00:00", "predicted_flow": 121.0}
    ],
    "generated_at": "2026-07-01T10:15:00"
  }
}
```

**Error (422)**：
```json
{"code": 422, "data": null, "message": "预测参数不合理：该路段历史数据不足，建议至少积累24小时数据"}
```

### 5.2 GET /predict/accuracy

**Query**：`?section_id=1&days=7`

**Response (200)**：

## 7. 路径规划模块 `/api/v1/route`

### 7.1 POST /route/plan

```
计算最优路径
```

**Request**：
```json
{
  "origin_section_id": 1,
  "dest_section_id": 12,
  "criteria": "time"
}
```

| 字段 | 说明 |
|------|------|
| origin_section_id | 起点路段ID |
| dest_section_id | 终点路段ID |
| criteria | 优化目标：`time`(最短时间)/`distance`(最短距离)，默认time |

**Response (200)**：
```json
{
  "code": 200,
  "data": {
    "route_id": 42,
    "origin": {"section_id": 1, "name": "主干道-南北1"},
    "destination": {"section_id": 12, "name": "次干道-东西3"},
    "criteria": "time",
    "total_distance": 5.8,
    "estimated_time": 12,
    "path": [
      {"section_id": 1,  "name": "主干道-南北1", "distance": 2.5, "time": 5},
      {"section_id": 8,  "name": "次干道-南北2", "distance": 1.5, "time": 3},
      {"section_id": 12, "name": "次干道-东西3", "distance": 1.8, "time": 4}
    ],
    "created_at": "2026-07-01T10:15:00"
  }
}
```

**Error (422)**：
```json
{"code": 422, "data": null, "message": "无法找到从路段1到路段12的可达路径"}
```

## 9. WebSocket 事件协议（与D3-T04对齐确认）

### 服务端 → 客户端

| 事件名 | Payload | 说明 |
|--------|---------|------|
| `warning:new` | `{warning_id, section_id, section_name, level, message, timestamp}` | 新预警触发 |
| `warning:update` | `{warning_id, is_resolved, resolved_at}` | 预警状态更新 |
| `warning:resolve` | `{warning_id, resolved_at}` | 预警已解除（广播） |
| `traffic:update` | `{section_id, vehicle_count, avg_speed, occupancy, level, timestamp}` | 实时路况更新 |

### 客户端 → 服务端

| 事件名 | Payload | 说明 |
|--------|---------|------|
| `subscribe:section` | `{section_id}` | 订阅某路段实时数据 |
| `unsubscribe:section` | `{section_id}` | 取消订阅 |
| `subscribe:all` | `{}` | 订阅全部路段 |
