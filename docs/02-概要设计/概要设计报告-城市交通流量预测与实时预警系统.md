## 摘要

本报告为「城市交通流量预测与实时预警系统」的概要设计文档，涵盖系统总体架构、模块划分、API接口设计、数据库设计、算法方案、前端架构、地图集成等核心设计内容。

系统采用**前后端分离 + MVC三层架构**，基于 **Flask + Vue 3 + SUMO + Scikit-learn** 技术栈，构建集微观交通仿真、短时流量预测、拥堵实时预警、最优路径规划于一体的智能交通管理平台。

本阶段共产出 **10份专项设计文档**，覆盖架构/算法/前端/地图/数据库/API/测试全维度，并通过 Agent-Judge 独立审查（D3阶段5/5 Approved）。

## 1. 总体架构设计

> 📄 详见：[总体架构设计与模块划分-20260701.md](总体架构设计与模块划分-20260701.md) (Agent-Lead, D3-T01, ✅ Approved)

### 1.1 架构模式

前后端分离 + MVC三层架构（路由层→服务层→数据层）。

### 1.2 技术栈

| 层 | 核心技术 |
|----|---------|
| 前端 | Vue 3 + Vite + Element Plus + ECharts + 高德地图JS API 2.0 |
| 后端 | Flask 3.x + SQLAlchemy + Celery + Redis |
| 算法 | SUMO 1.19+ + Scikit-learn(KNN+RF) + Pandas |
| 数据库 | MySQL 8.x |
| 实时通信 | Flask-SocketIO |

### 1.3 关键架构决策

| ADR | 决策 | 理由 |
|-----|------|------|
| ADR-001 | Flask而非Django | 轻量灵活，14天工期 |
| ADR-002 | KNN+RF双模型 | 快速迭代，可解释 |
| ADR-003 | 前后端分离 | 并行开发 |
| ADR-004 | WebSocket推送 | 秒级预警延迟 |
| ADR-005 | 五Agent协作 | 角色分离，独立审查 |

## 2. 模块划分

### 2.1 六大功能模块

| 模块 | 负责Agent | 核心功能 |
|------|----------|---------|
| M1 数据采集 | Agent-Algorithm | SUMO仿真 + 检测器数据读取 |
| M2 流量预测 | Agent-Algorithm | KNN+RF短时预测 + 模型评估 |
| M3 实时监控 | FE-Main + FE-Map | 地图路况 + ECharts图表 |
| M4 拥堵预警 | Leader + FE-Map | WebSocket推送 + 规则引擎 |
| M5 路径规划 | Agent-Lead | Dijkstra最优路径 |
| M6 系统管理 | Agent-Lead | JWT认证 + 日志管理 |

### 2.2 模块间数据流

```
SUMO仿真 → 检测器数据 → 预处理 → KNN/RF训练 → predict API
                                              ↓
                               ECharts图表 ← 预测JSON ← 预警检查
                                              ↓
                                        WebSocket推送 → 弹窗
```

## 3. API接口设计

> 📄 详见：[API详细接口规范-20260701.md](API详细接口规范-20260701.md) (Agent-Lead, D4-T01)
> 📄 详见：[模型接口详细规范-20260701.md](模型接口详细规范-20260701.md) (Agent-Algorithm, D4-T02)

### 3.1 API路由总览

| 模块 | 前缀 | 端点示例 | 认证 |
|------|------|---------|------|
| 认证 | `/api/v1/auth` | POST login, POST register, POST refresh | 部分 |
| 路段 | `/api/v1/sections` | GET list, POST, PUT, DELETE | 是 |
| 流量 | `/api/v1/traffic` | GET current, GET history | 是 |
| 预测 | `/api/v1/predict` | GET forecast, GET accuracy | 是 |
| 预警 | `/api/v1/warning` | GET list, PUT resolve, GET/PUT rules | 是 |
| 路径 | `/api/v1/route` | POST plan, GET/{id} | 是 |
| 统计 | `/api/v1/stats` | GET dashboard, GET daily_report | admin |

### 3.2 统一返回格式

```json
{"code": 200, "data": {...}, "message": "ok"}
```

分页：`{"items":[...], "total":100, "page":1, "page_size":20, "total_pages":5}`

### 3.3 错误码

`200/201/400/401/403/404/409/422/500`

## 4. 数据库设计

> 📄 详见：[数据库设计与E-R图-20260701.md](数据库设计与E-R图-20260701.md) (Agent-Test-Docs, D3-T05, ✅ Approved)

### 4.1 核心实体（8张表）

| 表名 | 关键字段 | 预估数据量 |
|------|---------|-----------|
| `users` | id, username, password_hash, role | <100 |
| `traffic_sections` | id, name, capacity, coordinates(JSON) | 24 |
| `traffic_detectors` | id, section_id(FK), type, position | 40 |
| `traffic_records` | id, section_id, detector_id, vehicle_count, avg_speed, occupancy, timestamp | ~3840/天 |
| `prediction_results` | id, section_id, model_type, predicted_flow, confidence_* | ~960/天 |
| `warning_events` | id, section_id, level, message, is_resolved | 不定 |
| `route_records` | id, user_id, origin→dest, route_path(JSON) | 不定 |
| `system_logs` | id, user_id, action, details(JSON), ip_address | 不定 |

### 4.2 实体关系

```
users 1──N route_records, system_logs
traffic_sections 1──N detectors, records, predictions, warnings
traffic_detectors 1──N traffic_records
```

## 5. 算法模块设计

> 📄 详见：[算法模块设计-20260701.md](算法模块设计-20260701.md) (Agent-Algorithm, D3-T02, ✅ Approved)

### 5.1 SUMO仿真方案

- 路网：24路段、12交叉口、2信号灯
- 车流：3时段（早高峰/平峰/晚高峰），车型比例85/8/7
- 检测器：40个E2线圈，15min采样间隔

### 5.2 数据管道

```
SUMO仿真 → XML解析 → 缺失值/异常值处理 → 特征工程(滞后+滚动+时间) → 归一化 → 训练/预测
```

### 5.3 双模型策略

| 模型 | 超参数 | 评估指标(期望) |
|------|--------|--------------|
| KNN回归 | K=[3,5,7,10,15], distance/uniform | MAE<15, R²>0.75 |
| 随机森林 | n=[100,200], depth=[10,15,None] | MAE<10, R²>0.82 |

策略模式（BaseTrafficPredictor）确保模型可插拔。

## 6. 前端架构设计

> 📄 详见：[前端架构与路由设计-20260701.md](前端架构与路由设计-20260701.md) (Agent-Frontend-Main, D3-T03, ✅ Approved)

### 6.1 页面结构

9个页面 + 15+组件，路由懒加载分chunk。

| 路由 | 页面 | 首屏chunk |
|------|------|----------|
| `/login`, `/register` | 登录/注册 | ~20KB |
| `/dashboard` | 首页 | ~30KB |
| `/traffic` | 实时路况 | ~80KB |
| `/prediction` | 预测看板 | ~150KB |
| `/warnings` | 预警管理 | ~25KB |
| `/route-plan` | 路径规划 | ~30KB |
| `/admin/*` | 管理 | ~20KB |

### 6.2 状态管理

3个Pinia Store：`useUserStore`(认证) + `useTrafficStore`(数据) + `useWarningStore`(预警)

### 6.3 API层

Axios JWT自动注入 + 401自动跳转 + 7个API模块封装 + Mock数据（后端未就绪时前端独立开发）

## 8. WebSocket实时通信

> 📄 详见：[WebSocket消息格式规范-20260701.md](WebSocket消息格式规范-20260701.md) (Agent-Frontend-Map, D4-T04)

### 8.1 协议

Socket.IO单例客户端，退避重连(1s→30s)。

| 方向 | 事件 | 触发条件 |
|------|------|---------|
| S→C | `warning:new` | 新预警触发 |
| S→C | `traffic:update` | 每15s推送路况 |
| C→S | `subscribe:section` | 用户点击路段 |

### 8.2 消息格式

全部使用TypeScript接口定义，6个事件完整Schema。
