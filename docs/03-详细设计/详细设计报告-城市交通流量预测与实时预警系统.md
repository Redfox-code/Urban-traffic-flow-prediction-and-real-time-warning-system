### 2.3 数据库模型（8个实体）

```python

User, TrafficSection, TrafficDetector, TrafficRecord,
PredictionResult, WarningEvent, RouteRecord, SystemLog
```

- ORM：SQLAlchemy 2.x
- 迁移：Flask-Migrate
- 测试：SQLite 内存库

### 2.4 核心算法

**Dijkstra 路径规划**（D9-T02, D11改进）：
- 基于 haversine 坐标距离构建邻接表
- 1km内视为相邻路段
- heapq 优先队列优化 → O((V+E)logV)

**预警规则引擎**（D8-T03）：
- occupancy ≥ 85% → WARNING
- occupancy ≥ 95% → CRITICAL

## 4. 算法模块实现

### 4.1 SUMO 仿真

```
algorithm/sumo/
├── city_flows.rou.xml    # 3时段(早/平/晚)+3车型(85/8/7)
├── detectors.add.xml     # 28个E2检测器(15min采样)
├── config.sumocfg        # 6h仿真配置
└── (city_network.net.xml) # netgenerate生成
```

### 4.2 数据管道

```python
parse_e2_output(xml) → clean_data(df) → build_features(df) → normalize(df)
```

- 缺失值：前值填充 → 中位数填充
- 异常值：IQR方法 (Q1-1.5×IQR, Q3+1.5×IQR)
- 特征：时间 + 滞后(t-1,t-2,t-3) + 滚动统计(4步=1h)

### 4.3 双模型

| 模型 | 超参数搜索 | 评估方法 |
|------|----------|---------|
| KNN | K=[3,5,7,10,15], metric=[euclidean,manhattan] | GridSearchCV 5-fold |
| RF | n=[100,200], depth=[10,15,None] | GridSearchCV 5-fold |

策略模式：BaseTrafficPredictor → KNNPredictor / RFPredictor

## 6. 部署说明

### 6.1 后端启动

```bash
cd backend
pip install -r requirements.txt
flask --app run.py run --port 5000
```

### 6.2 前端启动

```bash
cd frontend
npm install
npm run dev
```

### 6.3 算法运行

```bash
cd algorithm
python run_simulation.py generate  # 生成路网
python run_simulation.py run       # 运行仿真
python preprocessor.py data/raw/e2_output.xml  # 预处理
python train.py data/raw/e2_output.xml         # 训练模型
```
