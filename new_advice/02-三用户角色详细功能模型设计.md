# 城市交通流量预测与实时预警系统

## 三用户角色详细功能模型设计

> 本文档从交通管理员、数据分析员、普通出行者三个核心角色的第一视角出发，详细设计每个角色的功能页面、交互流程、数据需求、个性化策略和后端支撑逻辑。

---

## 一、角色全景对比

| 维度 | 交通管理员 | 数据分析员 | 普通出行者 |
|------|-----------|-----------|-----------|
| **使用场景** | 指挥中心大屏，持续监控 | 办公室PC，阶段性深度分析 | 手机/PC，出行前快速决策 |
| **使用频率** | 全时段实时在线 | 每天1-3次深度使用 | 每天1-2次（通勤前后） |
| **单次使用时长** | 持续数小时 | 30-60分钟 | 1-3分钟 |
| **信息密度需求** | 极高：一屏展示全域态势 | 高：多维度交叉分析 | 低：核心结论优先 |
| **操作复杂度** | 中等：需快速响应和决策 | 高：需参数配置和数据探索 | 低：2-3步完成主要任务 |
| **关键指标** | 拥堵路段数、预警处置率 | 模型MAPE、预测偏差分布 | 预计到达时间、路线拥堵度 |
| **消息关注点** | 实时预警推送 | 模型精度告警 | 常走路线拥堵提醒 |
| **登录状态** | 必须登录 | 必须登录 | 路径规划免登录，个性化需登录 |

## 二、交通管理员 — 功能详细设计

### 2.1 角色画像

- **岗位**: 城市交通管理中心值班人员
- **工作模式**: 轮班制，每班次2-4小时持续监控
- **核心痛点**: （1）信息过载，需要快速定位真正的异常点；（2）预警太多导致"狼来了"效应，需要智能降噪；（3）从"发现问题"到"做出决策"的链路太长
- **设计原则**: （1）异常驱动而非数据驱动：正常状态不打扰，异常状态主动推送；（2）分级降噪：同一路段的连续预警自动合并；（3）操作闭环：从看到预警到完成处置不超过3次点击

### 2.4 信号优化建议 `/admin/signal-optimization`

**页面定位**: 从预警处置中"调整信号配时"链路深入，也可独立访问。

**左右分栏布局**:
- 左侧(350px): 路口列表，按优化潜力降序排列，每项显示: 路口名/优化潜力百分比/当前延误时长/简要建议，点击切换右侧详情
- 右侧: 路口详情面板
  - 路口拓扑图(简化的交叉口示意图，标注各进口道方向)
  - 当前配时 vs 建议配时对比表(周期时长、各相位绿灯秒数、改善百分比)
  - 预估效果: 通行效率提升%、平均延误减少min、CO2减排量
  - 操作按钮: [应用建议] 通过TraCI写入SUMO验证 / [仿真验证] 在SUMO中运行1个周期对比 / [一键优化全部] 批量计算所有路口
- 下方: 优化历史记录表格

**Webster配时公式**:
```
C_opt = (1.5 * L + 5) / (1 - Y)
L = 总损失时间(每相位3秒黄灯+2秒全红)
Y = 各相位最大流量比(y = 流量/饱和流量)之和
有效绿灯时间 = C_opt - L
各相位绿灯分配: g_i = (y_i / Y) * (C_opt - L)
```

## 三、数据分析员 — 功能详细设计

### 3.1 角色画像

- **岗位**: 交通数据分析师
- **工作模式**: 每日1-3次深度使用，阶段性输出分析报告
- **核心痛点**: （1）模型精度不可见：不知道什么时候模型开始"老化"；（2）调参试错成本高：每次重训练需要时间，缺乏快速A/B对比；（3）分析结果与管理员的决策需求脱节：做了分析但无法直接变成可落地的建议
- **设计原则**: （1）模型健康度可视化：精度趋势自动追踪，异常自动告警；（2）What-If先行：做任何真实调整前，先在仿真环境验证；（3）分析即报告：所有分析视图可一键导出为报告片段

### 3.2 模型管理 `/analyst/models`

**页面定位**: 分析员默认首页，模型健康度一览。

**布局**:

- 顶部: 下次自动训练倒计时(Celery定时任务状态)
- 模型状态卡片(左右并排两张):
  - KNN卡片: 状态指示灯(绿/黄/红) + 最后训练时间 + K值 + MAE/RMSE/MAPE指标 + [重新训练]按钮
  - 随机森林卡片: 状态指示灯 + 最后训练时间 + 树数量 + MAE/RMSE/MAPE指标 + [重新训练][参数调优]按钮
  - 状态灯逻辑: 绿灯=精度稳定或上升; 黄灯=连续2天MAPE上升; 红灯=MAPE超过告警阈值(默认>15%)
- 精度趋势图(ECharts双折线): 过去7天MAPE变化，红色虚线标注告警阈值
- 训练历史表格: 时间/模型类型/MAE/RMSE/MAPE/参数/操作人

**参数调优面板**(点击RF卡片的[调优]打开):
- 随机森林可调参数: n_estimators(滑块50-300, 默认100), max_depth(滑块5-30, 默认10), min_samples_split(滑块2-20, 默认2)
- KNN可调参数: K值(滑块1-20, 默认5), 距离度量(下拉: Euclidean/Manhattan), 权重方式(下拉: uniform/distance)
- [应用并重新训练] -> Celery异步执行 -> WebSocket通知完成

### 3.3 拥堵传播分析 `/analyst/propagation`

**页面定位**: 数据深度分析工具，按需使用。

**左右分栏布局**:
- 左侧控制面板(350px): 起始路段(Select搜索) + 传播步数(滑块1-5跳) + 预测时间窗口(15/30/60分钟) + 置信度阈值(50%-90%) + [开始分析]
- 右侧可视化区域:
  - 上半部分: 路网拓扑传播树图(根节点=起始路段，子节点=受影响路段，边标注传播概率+延迟，树深度=步数)
  - 下半部分: 高德地图叠加箭头图层(渐变箭头颜色表示传播概率高低，方向表示传播方向)
- 下方: 传播链数据表格(源路段/目标路段/传播延迟/传播概率/置信度)
- 底部: 历史传播事件回放组件(选择历史事件 -> 播放进度条控制)

**传播算法设计**:
```
输入: 起始路段s0, 传播步数d, 预测时间窗口t, 置信度阈值p
输出: 传播树T

算法步骤:
1. 从路网邻接矩阵G提取s0的相邻路段集合N(s0)
2. 对每个相邻路段n:
   (a) 获取n的当前拥堵等级
   (b) 查历史数据中「s0拥堵且30min内n也拥堵」的频次f
   (c) 传播概率 = f / (s0拥堵总次数)
   (d) 传播延迟 = 历史中s0到n拥堵的时间差中位数
3. 对传播概率>阈值的边，递归执行步骤2
4. 剪枝条件: 传播概率<阈值 或 深度>d
5. 输出传播树+每条边的(概率,延迟)
```

## 四、普通出行者 — 功能详细设计

### 4.1 角色画像

- **场景一: 通勤白领** — 每天早8:00出门，晚18:00回家，固定OD对，对路线非常熟悉但需要提前预知异常
- **场景二: 临时出行者** — 偶尔使用，打开即用用完即走，不注册不登录
- **场景三: 外地访客** — 不熟悉路网，依赖导航系统，对路况信息敏感
- **核心痛点**: （1）不知道什么时候出门最合适，只能凭经验；（2）出发前无法预判路况，路上才发现拥堵；（3）遇到突发拥堵不知道该怎么办，缺乏备选方案
- **设计原则**: （1）免登录可用核心功能（路径规划），降低使用门槛；（2）登录后越用越智能（画像学习），让系统适应人；（3）2步完成核心操作：选起终点 -> 看结果，不设多余步骤；（4）异常主动告知：已规划的路线如果突然变堵，主动推送通知

**交互流程详解**:

1. 打开页面 -> 请求GPS权限(浏览器原生API) -> 获取位置填入起点
   - 若拒绝/不可用 -> 起点为空，提示"请在地图上点击选择起点"
2. 输入/选择终点(两种方式):
   - 方式A: 在搜索框输入地址/POI名称 -> 高德地图POI搜索API -> 下拉建议列表
   - 方式B: 在地图上直接长按/右键点击选点
3. 选择出发时间:
   - "现在出发": 使用实时路况数据计算
   - "指定时间": 弹出时间选择器(精确到分) -> 使用预测模型数据计算
4. 点击[开始规划路线] -> 调用 POST /api/route/plan
   - 请求体: { origin: {lat,lng}, destination: {lat,lng}, depart_time: "2026-07-12T08:00:00", use_prediction: true }
   - 响应体: { routes: [ { id, segments:[{section_id, name, congestion_level, travel_time}], total_time, total_distance, departure_time, arrival_time } ] }
5. 路线结果展示:
   - 地图同步切换路线绘制(不同颜色)
   - 默认选中推荐路线(展开详情)
   - 点击切换路线 -> 地图同步高亮
   - 点击某路段 -> 查看该路段实时/预测详情弹窗
6. [保存为常用路线]:
   - 未登录 -> 弹出登录引导框
   - 已登录 -> 直接保存(可自定义命名)
7. [开始导航] -> 根据设备类型: 手机端调起高德地图App导航; PC端复制链接或显示路线文字描述

**路线排序算法**:
```
输入: origin, destination, depart_time
输出: 3条路线(推荐/备选/最短距离参考)

步骤:
1. Dijkstra计算最短距离路径 -> 路线C(最短距离)
2. Dijkstra计算最短时间路径(权重=当前/预测行程时间) -> 路线A(推荐)
3. 从路线A中随机排除1-2条拥堵路段 -> 重新计算 -> 路线B(备选)
4. 排序: A(推荐) > B(备选) > C(距离参考)

时间权重计算:
  travel_time = 路段长度(m) / 预测速度(m/s)
  预测速度 = f(预测流量, 路段容量限速, 历史速度-流量拟合曲线)
  拥堵等级 = 根据预测速度分四档: >40畅通, 25-40缓行, 15-25拥堵, <15严重拥堵
```

**常用路线自动识别算法**:
```
输入: 用户所有路径规划查询记录(最近30天)
输出: 常用路线列表(OD对 + 出行时段 + 频次)

步骤:
1. 对查询记录按(origin_lat, origin_lng, dest_lat, dest_lng)分组
   容差: 经纬度差<0.001(约100m)视为同一OD对
2. 对每组OD对:
   (a) 统计查询频次 f
   (b) 使用K-means(k=1或2)聚类出发时间，识别早晚高峰模式
   (c) 给路线自动打标签:
       - depart_hour 在 7:00-9:30 + 工作日 -> "上班路线"
       - depart_hour 在 17:00-19:30 + 工作日 -> "回家路线"
       - 周末 -> "周末出行"
       - depart_hour 在 9:31-16:59 -> "日常出行"
       - 其他 -> "常用路线N"
3. 筛选 f >= 3 的OD对(至少查询过3次)
4. 按频次降序排列，最多展示5条
5. 定时(每天凌晨)重新计算更新
```

### 5.2 画像学习核心算法

**触发时机**: 每次用户执行路径规划查询后异步触发

```
函数: update_travel_profile(user_id, origin, destination, depart_time)

步骤:
1. depart_hour = round(depart_time.hour + depart_time.minute/60)
2. 查是否存在匹配的OD对
   (容差: |lat1-lat2|<0.001 AND |lng1-lng2|<0.001, 约100m)
3. 若存在:
   (a) frequency += 1
   (b) 指数加权移动平均更新 depart_hour_avg:
       new_avg = 0.7 * old_avg + 0.3 * current_hour
       解释: 给历史值70%权重，当前值30%权重，
             保证画像稳定但能适应出行习惯的缓慢变化
   (c) 更新 last_used_at = NOW()
   (d) 更新 depart_dayofweek = depart_time.weekday()
4. 若不存在:
   (a) 创建新记录: frequency=1, depart_hour_avg=current_hour
5. 若 frequency >= 3 AND route_label 为空:
   (a) 自动打标签(基于时段+星期规则)
6. 异步: 重新排序该用户的常用路线列表(Redis缓存更新)
```

### 5.3 用户分群与差异化策略

| 用户分群 | 识别条件 | 推送策略 | 界面策略 |

### 7.2 异常状态处理

| 异常场景 | 用户感知 | 系统处理 |
|---------|---------|---------|
| SUMO仿真中断 | 地图路况层显示"数据更新时间: HH:MM" + 黄色提示条 | 自动重试连接，降级为最后一次有效数据 |
| 预测模型加载失败 | 路径规划回退为"仅使用实时路况" + 分析员端红色告警 | 降级方案: 使用历史同期均值替代预测值 |
| WebSocket断连 | 前端显示"连接断开，正在重连..." + 指数退避自动重试(1s,2s,4s,8s,最大30s) | 重连后补拉断连期间的消息 |
| GPS定位失败 | 起点输入框显示"无法获取位置，请手动输入或在地图上选点" | 不阻塞流程，允许手动输入 |
| 路径规划无结果 | 地图弹层"起终点之间无可通行路径，请检查是否选择了正确位置" | 不显示空列表，给出明确提示 |
| 模型训练超时 | 分析员端显示"训练任务超时(>10min)，已自动终止" + [重试] | Celery设置timeout=600s，超时自动kill |
