# 城市交通流量预测与实时预警系统 — 运行操作说明

> **适用平台**：Windows 10/11 | **Python**：3.10+ | **Node.js**：18+

---

## 一、环境准备

### 1.1 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

**验证**：`pip list | grep Flask` 应显示 Flask 3.x。

### 1.2 安装 Node.js 依赖

```bash
cd frontend
npm install
```

**验证**：`npm list vite` 应显示 vite 6.x。

### 1.3 安装 SUMO（可选，仅算法模块需要）

从 [sumo.dlr.de](https://sumo.dlr.de/docs/Downloads.php) 下载安装 SUMO 1.19+，确保 `sumo` 和 `netgenerate` 在 PATH 中。

```bash
sumo --version   # 验证安装
```

---

## 二、启动系统

### 2.1 启动后端（终端1）

```bash
cd backend
flask --app run.py run --port 5000
```

**预期输出**：
```
 * Serving Flask app 'run.py'
 * Running on http://127.0.0.1:5000
```

**测试后端是否正常**：

```bash
curl http://localhost:5000/api/v1/sections
# 应返回: {"code":200,"data":{"items":[],...},"message":"ok"}
```

### 2.2 启动前端（终端2）

```bash
cd frontend
npm run dev
```

**预期输出**：
```
  VITE v6.4.3  ready in 500 ms
  ➜  Local:   http://localhost:5173/
```

**打开浏览器** → `http://localhost:5173`

### 2.3 启动 SUMO 仿真（终端3，可选）

```bash
cd algorithm
python run_simulation.py generate   # 生成路网+检测器+车流
python run_simulation.py run        # 运行仿真
```

**预期输出**：
```
[SUMO] 生成路网...
[SUMO] 路网已生成: city_network.net.xml
[SUMO] 发现 20 条道路，自动生成检测器...
[SUMO] 运行仿真...
Step #0.00 ...
```

## 三、系统页面导航

### 3.1 登录页 — `/login`

- 首次访问自动跳转登录页
- 输入用户名密码 → 点击登录
- 无账号可点「注册」创建

**测试账号**（通过API注册或程序内注册）：
```
用户名: admin   密码: admin123
```

### 3.2 系统首页 — `/dashboard`

- 登录后默认进入
- 显示4个统计卡片：路段总数/检测器数/今日预警/预测精度

### 3.3 实时路况 — `/traffic`

- 高德地图大屏展示（暗色主题）
- 路段标注点标记
- 点击路段 → 右侧显示该路段实时数据
- WebSocket 实时推送路况更新

### 3.4 流量预测 — `/prediction`

- 选择路段 + 选择模型（RF/KNN）+ 点击预测
- 显示预测结果卡片：预测流量 + 置信区间
- ECharts 预测趋势图

### 3.5 预警管理 — `/warnings`

- 预警列表（筛选：全部/未解除/严重）
- 每条预警：路段名 + 等级标签 + 描述 + 时间
- 点击「解除」关闭预警

### 3.6 路径规划 — `/route-plan`

- 选择起点路段 → 选择终点路段 → 点击规划路径
- 显示总距离 + 预估时间
- 时间线展示途经路段

### 3.7 用户管理 — `/admin/users`（仅admin）

- 用户列表 + 添加新用户

### 3.8 系统日志 — `/admin/logs`（仅admin）

- 操作日志查看

## 四、API 测试

后端启动后，可直接用 curl 测试所有接口：

```bash
# 注册
curl -X POST http://localhost:5000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"pass","role":"analyst"}'

# 登录（获取Token）
curl -X POST http://localhost:5000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"pass"}'

# 用返回的Token访问需要认证的端点
TOKEN="<从登录返回中复制>"

# 路段列表
curl http://localhost:5000/api/v1/sections \
  -H "Authorization: Bearer $TOKEN"

# 流量预测
curl "http://localhost:5000/api/v1/predict/forecast?section_id=1&model=RF" \
  -H "Authorization: Bearer $TOKEN"

# 预警列表
curl http://localhost:5000/api/v1/warning/list \
  -H "Authorization: Bearer $TOKEN"

# 路径规划
curl -X POST http://localhost:5000/api/v1/route/plan \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"origin_section_id":1,"dest_section_id":2}'

# Dashboard统计
curl http://localhost:5000/api/v1/stats/dashboard \
  -H "Authorization: Bearer $TOKEN"
```

## 六、常见问题

### Q1: `flask --app run.py run` 报 import 错误

```bash
cd backend && pip install -r requirements.txt
```

### Q2: `npm run dev` 报 module 找不到

```bash
cd frontend && rm -rf node_modules && npm install
```

### Q3: 地图不显示

高德地图需要有效的 Key。前往 [高德开放平台](https://lbs.amap.com) 申请，填入 `frontend/.env.development`：

```
VITE_AMAP_KEY=你的Key
```

### Q4: 预测接口返回 503

模型文件未训练。运行 `cd algorithm && python train.py` 生成模型文件后，预测接口才能正常工作。

### Q5: SUMO 报错 `command not found`

SUMO 未安装或不在 PATH 中。从 [sumo.dlr.de](https://sumo.dlr.de/docs/Downloads.php) 下载安装。
