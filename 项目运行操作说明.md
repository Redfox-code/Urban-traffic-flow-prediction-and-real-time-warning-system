# 城市交通流量预测与实时预警系统 — 运行操作说明

> **适用平台**：Windows 10/11 | **Python**：3.10+ | **Node.js**：18+
> **最后更新**：2026-07-02（系统测试通过，17/17 pytest passed）

---

## 一、环境准备

### 1.1 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

**验证**：`pip list | grep Flask` 应显示 Flask 3.x。

### 1.2 安装 Node.js 依赖

```bash
cd frontend
npm install
```

**验证**：`npm list vite` 应显示 vite 6.x。

### 1.3 安装 SUMO（可选，仅算法模块需要）

从 [sumo.dlr.de](https://sumo.dlr.de/docs/Downloads.php) 下载安装 SUMO 1.19+，确保 `sumo` 和 `netgenerate` 在 PATH 中。

```bash
sumo --version   # 验证安装
```

### 1.4 初始化数据库

```bash
cd backend
python seed_data.py
```

**预期输出**：`Seed完成: 2用户, 24路段, 36检测器`

**预置账号**：
| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 管理员 |
| analyst | analyst123 | 分析员 |

## 二、启动系统

### 2.1 启动后端（终端1）

```bash
cd backend
flask --app run.py run --port 5000
```

**预期输出**：
```
 * Serving Flask app 'run.py'
 * Running on http://127.0.0.1:5000
```

### 2.2 启动前端（终端2）

```bash
cd frontend
npm run dev
```

**预期输出**：
```
  VITE v6.4.3  ready in 500 ms
  ➜  Local:   http://localhost:5173/
```

**打开浏览器** → `http://localhost:5173` → 使用 admin/admin123 登录。

### 2.3 运行 SUMO 仿真并导入数据（终端3，可选）

```bash
cd algorithm
python run_simulation.py generate   # 生成路网+检测器+车流
python run_simulation.py run        # 运行仿真（约30秒）
python import_sumo_data.py          # 导入仿真数据到数据库
```

> **注意**：导入前请先关闭 Flask（Ctrl+C），导入后重新启动。

## 四、系统页面导航

### 4.1 登录页 — `/login`

- 首次访问自动跳转登录页
- 输入 admin/admin123 → 点击登录

### 4.2 系统首页 — `/dashboard`

- 4个统计卡片：路段总数(24) / 检测器数(36) / 今日预警 / 预测精度
- 卡片带渐变图标 + 悬浮动画
- 数据来源：`statsApi.getDashboard()`

### 4.3 实时路况 — `/traffic`

- 左侧高德地图（暗色主题）+ 右侧路段列表 + 数据面板
- 24条路段各有 TrafficBadge 标记（畅通🟢/缓行🟡/拥堵🟠/严重🔴）
- 点击路段 → 右侧面板显示：车流量/平均速度/占有率/通行能力/限速
- 页面加载时自动获取全部路段路况
- 地图需配置 `VITE_AMAP_KEY`（见 Q3）

### 4.4 流量预测 — `/prediction`

- 选择路段 + 模型（RF/KNN）+ 预测窗口（5/15/30分钟）
- 点击「开始预测」→ 显示预测流量 + 置信区间 + 预测序列卡片
- 无真实模型时自动使用 mock 预测（标记「模拟」标签）
- 数据来源：`predictionApi.getForecast()`

### 4.5 预警管理 — `/warnings`

- 筛选：全部 / 未解除 / 严重
- 每条预警：路段 + 等级标签 + 描述 + 时间 + 解除按钮
- 数据来源：`warningApi.getList()`

### 4.6 路径规划 — `/route-plan`

- 起点路段 → 终点路段 → 点击「规划路径」
- 显示总距离(km) + 预估时间(min) + 路径时间线
- 使用 Dijkstra + haversine 坐标距离算法
- 数据来源：`routeApi.plan()`

### 4.7 用户管理 — `/admin/users`（仅admin）

- 用户列表 + 添加新用户

### 4.8 系统日志 — `/admin/logs`（仅admin）

- 操作日志查看

## 五、API 测试

后端启动后，可直接用 curl 测试所有接口：

```bash
# 登录（获取Token）
curl -X POST http://localhost:5000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 复制返回的 token，设置变量
TOKEN="<token>"

# 路段列表
curl http://localhost:5000/api/v1/sections -H "Authorization: Bearer $TOKEN"

# 实时路况
curl "http://localhost:5000/api/v1/traffic/current?section_id=1" -H "Authorization: Bearer $TOKEN"

# 流量预测
curl "http://localhost:5000/api/v1/predict/forecast?section_id=1&model=RF" -H "Authorization: Bearer $TOKEN"

# 预警列表
curl http://localhost:5000/api/v1/warning/list -H "Authorization: Bearer $TOKEN"

# 路径规划
curl -X POST http://localhost:5000/api/v1/route/plan \
  -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" \
  -d '{"origin_section_id":1,"dest_section_id":2}'

# Dashboard
curl http://localhost:5000/api/v1/stats/dashboard -H "Authorization: Bearer $TOKEN"

# 历史流量
curl "http://localhost:5000/api/v1/traffic/history?section_id=1" -H "Authorization: Bearer $TOKEN"
```

> 所有端点（除 login/register）均需 JWT 认证，未认证返回 401。

## 七、常见问题

### Q1: `flask --app run.py run` 报 import 错误

```bash
cd backend && pip install -r requirements.txt
```

### Q2: `npm run dev` 报 module 找不到

```bash
cd frontend && rm -rf node_modules && npm install
```

### Q3: 地图不显示

1. 前往 [高德开放平台](https://lbs.amap.com) 申请免费 Key
2. 填入 `frontend/.env.development`（**不要加引号！**）：

```
VITE_AMAP_KEY=a7e006e65af936c9e57abc52fff9b826    ← 正确：无引号
VITE_AMAP_KEY='a7e00...'                          ← 错误：有引号
```

3. **重启前端**（`.env` 修改需要重启 Vite）

### Q4: 预测功能显示「模拟」标签

正常行为。系统在没有真实训练模型时自动使用 mock 数据。训练模型后标签会消失：
```bash
cd algorithm && python train.py data/raw/e2_output.xml
```

### Q5: SUMO 报错 `command not found`

SUMO 未安装或不在 PATH 中。从 [sumo.dlr.de](https://sumo.dlr.de/docs/Downloads.php) 下载安装。

### Q6: 路段列表一直显示「加载中」

1. 确认后端已启动：`curl http://localhost:5000/api/v1/sections -H "Authorization: Bearer <TOKEN>"`
2. 确认已运行过 `python seed_data.py`（数据库有数据）

### Q7: `NOT NULL constraint failed: traffic_records.id`

SQLite BigInteger + autoincrement 兼容性问题。已修复（BigInteger→Integer）。如仍有问题：
```bash
del backend\instance\dev.db
cd backend && python seed_data.py
```

### Q8: `dev.db is being used by another process`

Flask 正在运行占用数据库。先 Ctrl+C 停 Flask，再执行操作。
