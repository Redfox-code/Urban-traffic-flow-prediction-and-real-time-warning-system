# 系统测试工作流 — 端到端验证+自动返工

> **触发指令**：对 Claude 说「开始系统测试」
> **适用场景**：代码开发完成后，系统性验证后端+算法+前端+测试全部可运行
> **循环机制**：发现Bug → 分配给对应Agent → Agent修复 → 重测 → 直到全部通过

---

## 一、工作流概览

```
Phase 1: 后端测试 → Phase 2: 算法测试 → Phase 3: 前端测试 → Phase 4: pytest
     │                  │                   │                   │
     ▼                  ▼                   ▼                   ▼
  报错?→Bug→Lead   报错?→Bug→Algo    报错?→Bug→FE-Main   失败?→Bug→对应Agent
     │                  │                   │                   │
     └──修复──重测──┘   └──修复──重测──┘   └──修复──重测──┘   └──修复──重测──┘
```

每个Bug修复后必须重测该Phase，直到该Phase全部通过才进入下一Phase。

```
开始系统测试
```

## 三、Phase 1：后端测试

### 步骤 1.1：安装依赖

```bash
cd backend && pip install -r requirements.txt
```

**报错** → 建Bug `BUG-BE-01` → 分配给 **Agent-Lead**。

### 步骤 1.2：启动 Flask

```bash
cd backend && flask --app run.py run --port 5000 &
sleep 2
curl http://localhost:5000/api/v1/sections
```

**报错**（import错误/启动异常/500）→ 建Bug → 分配给 **Agent-Lead**。

### 步骤 1.3：测试认证端点

```bash
# 注册
curl -X POST http://localhost:5000/api/v1/auth/register -H "Content-Type: application/json" -d '{"username":"test","password":"pass","role":"analyst"}'
# 登录
curl -X POST http://localhost:5000/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"test","password":"pass"}'
```

**报错/非200** → 建Bug `BUG-BE-{XX}` → 分配给 **Agent-Lead**。

### 步骤 1.4：测试其他端点

```bash
# 路段列表
curl http://localhost:5000/api/v1/sections -H "Authorization: Bearer <TOKEN>"
# 预测
curl "http://localhost:5000/api/v1/predict/forecast?section_id=1" -H "Authorization: Bearer <TOKEN>"
# 预警
curl http://localhost:5000/api/v1/warning/list -H "Authorization: Bearer <TOKEN>"
# 路径规划
curl -X POST http://localhost:5000/api/v1/route/plan -H "Content-Type: application/json" -H "Authorization: Bearer <TOKEN>" -d '{"origin_section_id":1,"dest_section_id":2}'
# Dashboard
curl http://localhost:5000/api/v1/stats/dashboard -H "Authorization: Bearer <TOKEN>"
```

**报错** → 建Bug：
- auth/sections/stats/warning/route 端点 → 分配给 **Agent-Lead**
- traffic/predict 端点 → 分配给 **Agent-Algorithm**

### Phase 1 通过标准

所有端点返回 200/201，无 500 错误，无 import 异常。

## 五、Phase 3：前端测试

## 七、Bug处理规范

### Bug创建格式（写入 task-board.md 🐛 Bugs区）

```
| BUG-XX-01 | [描述] | 🔴严重/🟡一般 | [发现阶段] | [分配给] | [错误日志摘要] | Open |
```

### Agent修复Bug时必须完成的四步

```
1. agent-logs: 🎯Bug接收 → 💭分析根因 → 📝修复内容 → ✅验证通过
2. handoff-queue: 修复完成后登记，通知下游可继续测试
3. decisions-log: 如果修复涉及设计变更，追加决策记录
4. run-log: 追加 "[时间] 🐛Bug修复 | BUG-XX-01 | [摘要]"
```

> ⚠️ 四文件全部更新才算Bug关闭。修复后必须重测验证。

### 唤醒Agent修复Bug的指令

```
Agent-[角色名]，修复BUG-XX-01：[简述]
```

